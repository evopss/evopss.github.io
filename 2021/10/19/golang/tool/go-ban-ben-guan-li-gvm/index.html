【Go版本管理】Gvm

gvm github地址：https://github.com/moovweb/gvm

gvm配置

gvm安装

如果使用的是zsh只需要将bash换成zsh

bash < <(curl -s -S -L https://raw.githubusercontent.com/moovweb/gvm/master/binscripts/gvm-installer)

使用gvm管理go版本

➜  ~ gvm help   
Usage: gvm [command]

Description:
  GVM is the Go Version Manager

Commands:
  version    - print the gvm version number
  get        - gets the latest code (for debugging)
  use        - select a go version to use (--default to set permanently)
  diff       - view changes to Go root
  help       - display this usage text
  implode    - completely remove gvm
  install    - install go versions
  uninstall  - uninstall go versions
  cross      - install go cross compilers
  linkthis   - link this directory into GOPATH
  list       - list installed go versions
  listall    - list available versions
  alias      - manage go version aliases
  pkgset     - manage go packages sets
  pkgenv     - edit the environment for a package set

  • gvm listall 命令显示了可以下载和编译的可用的 Go 版本

  • gvm use设置系统的go版本

    gvm use go1.16
gvm use go1.16 --default

  • gvm diff

  • gvm install <version>安装指定版本的go

    gvm install go1.16

  • gvm uninstall <version>卸载指定版本的go

  • gvm list列出当前环境中所有的go版本

    ➜  ~ gvm list                
gvm gos (installed)
   go1.15.7
=> go1.16
   go1.16.7
   go1.17

  • gvm alias

  • gvm pkgset

    GVM 通过使用pkgsets将项目的新目录附加到 Go 安装版本的默认 $GOPATH，很好地完成了项目之间包的管理和隔离。

    # global pkgset将默认使用该版本
➜  ~ echo $GOPATH    
/home/enderviggin/.gvm/pkgsets/go1.16/global
➜  ~ gvm pkgset create opsmb
➜  ~ gvm pkgset use opsmb           
Now using version go1.16@opsmb
➜  ~ gvm pkgset list
gvm go package sets (go1.16)
    global
=>  opsmb
➜  ~ echo $GOPATH
/home/enderviggin/.gvm/pkgsets/go1.16/opsmb:/home/enderviggin/.gvm/pkgsets/go1.16/global

  • gvm pkgenv

  • gvm implode删除gvm和所有go版本和安装的包

Go环境变量

go env // 打印go的所有环境变量

go env GPPATH // 打印某个环境变量的值

常用环境变量

GOROOT

GOROOT表示go语言的安装目录，可以使用go env GOROOT查看。GOROOT/bin是go提供的工具链，如果需要全局使用，则需要将此配置在环境变量中。

# go 全局工具链配置(~/.zshrc, ~/.bashrc)
export GOROOT=$(go env GOROOT) # 默认GOROOT属于全局变量，可以不单独设置
export PATH=${PATH}:${GOROOT}/bin

GOPATH

GOPATH表示开发工作区，是用来存放源代码，测试文件，库静态文件，可执行文件。可以使用go env GOPATH查看。

可以通过修改GOPATH来更换工作区，这里推荐使用gvm pkgset来创建设置工作区

# GOPATH目录结构
➜  ~ tree $(go env GOPATH) -L 3 # 有缩减
├── bin            # 源代码编译后的可执行文件
│   ├── goctl 
├── overlay
│   ├── bin
│   └── lib
│       └── pkgconfig
├── pkg             # 库静态文件
│   ├── mod
│   │   ├── github.com
│   │   ├── go.etcd.io
│   └── sumdb
│       └── sum.golang.org
└── src             # 　开发的源代码文件
    └── github.com  # 包
        └── ender-viggin
# 设置工作区，虽然可以指定多个工作区，但是在使用go get安装包时，默认会安装到第一个工作区
export GOPATH=folder
export GOAPTH=folder1;folder2

GOBIN

GOBIN表示开发程序编译后的二进制命令的安装目录即GOPATH下面的bin目录。如果需要全局使用命令，需要配置在全局变量中

export GOBIN=${GOPATH}/bin

GOPROXY

设置go代理，提高下载速度

国内常用代理地址

goproxy: https://goproxy.io/zh/

七牛云： https://goproxy.cn

阿里云： https://mirrors.aliyun.com/goproxy/

代理设置方式

# Go 版本是 1.13 及以上
go env -w GO111MODULE=on
go env -w GOPROXY=https://goproxy.cn,direct

# 其他版本
export GO111MODULE=on
export GOPROXY=https://goproxy.cn

# 取消代理
go env -u GOPROXY

交叉编译

在一个平台上就能生成在其他平台上运行的代码。

GOOS和GOARCH

GOOS的默认值是我们当前的操作系统， GOARCH则表示CPU架构， 可以使用go env GOOS GOARCH查看当前系统的架构信息。
